## The Quant Library (libquant)
Проект с открытым исходным кодом, нацеленный на создание **быстрой кросс-платформенной DSP библиотеки не-реального времени**, которая поддерживает **мультипоточность**.

Текущий период разработки библиотеки - **Альфа**.

### Что может делать libquant?
- **Записывать и читать** следующие форматы:
    - Microsoft WAV файлы
      - PCM формат (16, 24, 32 бит)
      - IEEE-754 (floating-point) PCM формат (32, 64 бит)
- Она может применять следующие **эффекты**:
  - **Искажение** (клиппинг)
  - **Усиление** (и авто-усиление до максимума)
  - **Реверс**
  - **Хорус** (экспериментальный)
  - **Эффект течения** (плавный переход звука между каналами)
  - **Квантование** (поддерживает дизеринг)
  - **Инвертирование**
  - **Эффект затухания** (линейный или по синусу)
  - **Фильтр по среднему значению** (по медиане или среднему арифметическому)
  - **Эхо** (без эквалайзинга)
  - **Дилэй** (однократный эхо эффект)
  - **Глушение**
  - **Эффект ускорения** (без поднятия высоты тона)
  - **Смешивание с шумом** (псевдо-шум, использует rand())
  - **Микширование каналов**
  - **Манипуляции с разностью каналов** 
- Она может **синтезировать** некоторые волновые формы:
  - **Синусоиду**
  - **Меандр**
  - **Шум** (псевдо-шум, использует rand())
  - **Пила**
  - **Тишина**
  
### Ограничения
- Максимальное число каналов -  **(2^32)-1**
- Максимальная частота дискретизации - **(2^32)-1 Гц**
- Количество звуковых отсчётов хранится как *size_t*, его длина **зависит от платформы.**
- Каждый канал должен иметь одинаковое число звуковых отсчётов.

**Ремарка**: Когда вы используете OpenMP 2.0 (или старше), тогда в качестве итератора будет использоваться **long long**.

### Требования
- **C99** (**C17** рекомендуется) или **C++11** совместимый компилятор.
- Целевая архитектура должна:
    - Иметь <ins>8-битные байты</ins> (CHAR_BIT должен быть 8).
    - Соответствовать стандарту [<ins>IEEE-754</ins>](https://wikipedia.org/wiki/IEEE-754).
    - Использовать <ins>обратный или дополнительный код</ins>.
    - Быть либо <ins>Big-Endian, либо Little-Endian</ins>.
- <u>*для поддержки многопоточности*</u> | Компилятор с поддержкой **OpenMP 2.0** (OpenMP **3.0** рекомендуется).

**Ремарка**: Некоторые компиляторы, которые поддерживают C++11 **частично**, тоже могут скомпилировать libquant.
